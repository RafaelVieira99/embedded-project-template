# Contributing

This repository is part of an embedded systems portfolio project.

## Development Rules

Before making changes:

1. Check the open issues.
2. Create or select an issue for the work.
3. Keep commits small and focused.
4. Follow the coding style in `docs/coding-style.md`.
5. Format C/C++ code using `.clang-format`.

## Commit Style

Use clear commit messages.

Examples:

```text
Add UART CLI parser
Implement I2C sensor timeout handling
Document project requirements
Add hardware-in-the-loop test skeleton
```

When relevant, reference the issue number:

```text
Add UART CLI parser (#8)
```

## Definition of Done

A task is only done when:

- The code builds.
- The behavior is tested.
- Documentation is updated if needed.
- Known limitations are written down.
- The related issue is updated or closed.
